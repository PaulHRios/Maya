## English

### What it is

Maya is a **privacy-first PWA baby tracker** built by (and for) a real family with a newborn.
It runs as a static site on GitHub Pages — no backend, no accounts database, no analytics —
yet both parents' phones stay in sync in real time through the family's own **private GitHub
repository used as a database**, with an AES-GCM-encrypted access token that only decrypts
with the family's login credentials.

### Features

- 🤱 **Feeding**: breast timers (left/right, pausable), expressed milk & formula in ml, per-parent authorship
- 💩 **Diapers**: one-tap logging, physical-diaper counter, and **on-device photo analysis** that suggests stool color & consistency with pediatric guidance
- 🥛 **Milk bank**: fridge/freezer inventory with a live tank gauge (green/yellow/red vs. a target computed from the baby's real intake); bottles auto-deduct, pumping adds back
- ⚡ **Guided pumping**: 6 methods (power pumping, Medela 2-phase, Stanford hands-on…), phase alerts, records with celebrations, production charts and an adaptive coaching card
- 🌙 **Sleep**: sleep/wake-window tracking with age-based traffic-light gauge, night-vigil logbook with timestamps and caregiver hand-offs, customizable bedtime routines with a guided runner, data-driven sleep tips
- 🤖 **Local health analysis**: hydration, feeding frequency/volume, weight trajectory and condition trends vs. the baby's *own baseline* — 100% on-device
- 🗓️ **Stage guidance**: one-time pop-ups as the baby enters each stage (colic season, teething, 4-month sleep regression…) with relief tips and shopping hints
- 🏆 **Daily challenges**: age-appropriate stimulation activities with timers, confetti, streaks and medals; extra tasks generated by active medical conditions
- 💚 **Mom's wellbeing**: Edinburgh Postnatal Depression Scale on a clinical screening schedule (bi-weekly → monthly through month 12), private to mom's phone, with real crisis/therapy resources
- 📅 **Appointments & meds**: medical appointments with native calendar alarms (`.ics`), daily medication check-offs, conditions can be marked as resolved 🎉
- 📄 **Pediatrician PDF**: charts + tables for any date range (feeds, diapers, sleep, bilirubin curves)
- 👶 **Up to 5 babies**, 4 color themes (incl. dark mode), weekly photo reminders, PDF export, offline-first

### Try it / sample data

Open the **[interactive demo](https://paulhrios.github.io/Maya/index.html?demo=1)** — it seeds a
synthetic baby ("Emma", 5 weeks old, 4 days of realistic data) entirely in `localStorage`.
Nothing is uploaded; the demo cannot touch real data. The product UI is Spanish (built for a
Mexican family); the demo landing page explains everything in English.

### Run your own

1. Fork this repo and enable **GitHub Pages** (deploy from branch `main`, root).
2. Create a **private** repo for your data (e.g. `baby-data`) and a fine-grained PAT with
   *Contents: read & write* on that repo only.
3. Edit the `CUENTAS` block in `js/store.js`: put SHA-256 hashes of your `email|password`
   credentials and your token encrypted per credential (see `descifrarToken` — AES-GCM key
   derived with PBKDF2-SHA256, 310k iterations). Never commit plaintext secrets.
4. Open the page on each phone → *Share → Add to Home Screen* → log in. Done: both phones sync.

### How it's built

Vanilla JS PWA (no build step) + Chart.js + jsPDF, vendored locally for offline use.
Data layer: append-friendly collections merged by `id`/`updatedAt` with tombstones —
conflict-free sync between devices over the GitHub Contents API. Service worker:
network-first shell with full offline fallback.

## Español

### Qué es

Maya es una **app de bebé (PWA) privada por diseño**, construida por (y para) una familia real
con una recién nacida. Corre como sitio estático en GitHub Pages — sin backend, sin base de
datos de cuentas, sin analytics — y aun así los teléfonos de ambos papás se sincronizan en vivo
usando su propio **repositorio privado de GitHub como base de datos**, con el token cifrado
(AES-GCM) que solo se abre con las credenciales de la familia.

### Funciones

- 🤱 **Alimentación**: timers por pecho (izq/der, con pausa), leche extraída y fórmula en ml, firma de quién registró
- 💩 **Pañales**: registro de un toque, contador de pañales gastados, y **análisis de foto en el dispositivo** que sugiere color y consistencia con guía pediátrica
- 🥛 **Banco de leche**: inventario refri/congelador con tanque visual por tercios; los biberones se descuentan solos y las extracciones suman
- ⚡ **Extracción guiada**: 6 métodos (power pumping, Medela 2 fases, manos activas…), avisos de fase, récords con celebración, gráficas y entrenadora de producción
- 🌙 **Sueño**: ventanas de despierto por edad con semáforo, bitácora de vigilias con relevos y horas, rutinas de dormir personalizables con modo guiado, y consejos con sus datos
- 🤖 **Análisis local de salud**: hidratación, tomas, peso y condiciones contra su propia línea base — 100% en el dispositivo
- 🗓️ **Guía por etapas**: avisos al entrar a cada etapa (cólicos, dientes, regresión de los 4 meses…) con alivios y sugerencias de compras
- 🏆 **Retos diarios**: estimulación por edad con timers, confeti, rachas y medallero; tareas extra según condiciones activas
- 💚 **Bienestar de mamá**: Escala de Edimburgo con calendario clínico de tamizaje (quincenal → mensual hasta el año), privada en su teléfono, con recursos de ayuda reales
- 📅 **Citas y medicamentos**: citas con alarmas en el calendario nativo (`.ics`), check diario de medicamentos, y condiciones que se marcan como superadas 🎉
- 📄 **PDF para el pediatra**, 👶 hasta 5 bebés, 4 temas de color con modo oscuro, foto semanal, y funcionamiento sin internet

### Pruébala / datos de ejemplo

Abre el **[demo interactivo](https://paulhrios.github.io/Maya/index.html?demo=1)**: se genera una
bebé sintética ("Emma", 5 semanas, 4 días de datos realistas) solo en tu navegador. Nada se sube
a ningún servidor y el demo no puede tocar datos reales.

### Monta la tuya

1. Haz fork y activa **GitHub Pages** (rama `main`, raíz).
2. Crea un repo **privado** para los datos y un token fine-grained con *Contents: read & write*
   solo para ese repo.
3. Edita el bloque `CUENTAS` de `js/store.js` con los hashes SHA-256 de tus credenciales
   (`correo|contraseña`) y el token cifrado por credencial. Nunca subas secretos en texto plano.
4. En cada teléfono: *Compartir → Agregar a pantalla de inicio* → iniciar sesión. Listo.
